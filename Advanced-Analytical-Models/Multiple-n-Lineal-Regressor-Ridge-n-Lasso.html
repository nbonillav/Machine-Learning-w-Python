Ejercicio 1¶

[50 puntos] En este ejercicio vamos a trabajar con los datos seguro_v1.csv, es un conjunto de datos de usuarios que han necesitado aplicar su seguro médico, donde se presentan distintas propiedades de cada individuo y el correspondiente costo del seguro médico. El objetivo es poder determinar o predecir el costo del seguro médico. El conjunto de datos tiene 1338 filas y 7 columnas, las cuales se explican a continuación.

Las variables de la tabla de datos son las siguientes:

  • edad: Edad del beneficiario principal.
  • sexo: Género del asegurado, femenino, masculino.
  • imc: índice de masa corporal (IMC), que permite comprender el peso corporal, ya sea alto o bajo en relación con la estatura. índice objetivo de peso corporal (kg/m²) utilizando la relación entre la estatura y el peso, idealmente entre 18,5 y 24,9.
  • hijos: Número de hijos cubiertos por el seguro médico / Número de dependientes.
  • fumador: Fumador
  • region: Zona residencial del beneficiario en EE. UU.: noreste, sureste, suroeste y noroeste.
  • costos: Costos médicos individuales facturados por el seguro médico (variable a predecir).

Realice lo siguiente:

Cuál de los modelos es mejor para estos datos y por qué?, recuerde que debe comparar contra el resumen numérico de la variable a predecir.

def modelado(X, y, var_categoricas, var_numericas, modelo_predictorio, test_size=0.15):
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=test_size)

    preprocesamiento = ColumnTransformer(
        transformers=[
            ('cat', OneHotEncoder(), var_categoricas),
            ('num', StandardScaler(), var_numericas)
    ])

    modelo = Pipeline(steps=[
        ('preprocesador', preprocesamiento),
        ('clasificador', modelo_predictorio)
    ])

    modelo.fit(X_train, y_train)
    pred = modelo.predict(X_test)

    rmse = np.sqrt(np.mean((y_test - pred) ** 2))
    mae = np.mean(np.abs(y_test - pred))
    er = np.sum(np.abs(y_test - pred)) / np.sum(np.abs(y_test))


    diccionario = {
        'Modelo': modelo,
        'Error Cuadrático Medio (RMSE)': rmse,
        'Error Absoluto Medio (MAE)': mae,
        'Error Relativo (ER)': er
    }

    return diccionario

LassoRegression¶

  • alpha: Constante que multiplica el término L2, controlando la fuerza de regularización. Alfa debe ser un flotante no negativo, es decir, en [0, inf). Cuando alfa = 0, el objetivo es equivalente a la regresión lineal, resuelto por el objeto LinearRegression. Por razones numéricas, no se recomienda usar alpha = 0 con el objeto Ridge. En su lugar, debe usar el objeto LinearRegression.
  • fit_interceptbool: Si se establece en falso, no se utilizará ningun intercepto en los cálculos (es decir, se espera que X e y estén centrados). default = True

RidgeRegression¶

  • alpha: Constante que multiplica el término L2, controlando la fuerza de regularización. Alfa debe ser un flotante no negativo, es decir, en [0, inf). Cuando alfa = 0, el objetivo es equivalente a la regresión lineal, resuelto por el objeto LinearRegression. Por razones numéricas, no se recomienda usar alpha = 0 con el objeto Ridge. En su lugar, debe usar el objeto LinearRegression.
  • fit_interceptbool: Si se establece en falso, no se utilizará ningun intercepto en los cálculos (es decir, se espera que X e y estén centrados). `default = True
  • solver: {auto, svd, cholesky, lsqr, sparse_cg, sag, saga, lbfgs}, default='auto' Solver para usar en las rutinas computacionales:
  • auto: elige el solucionador automáticamente según el tipo de datos.
  • svd: utiliza una Descomposición de valor singular de X para calcular los coeficientes de Ridge. Es el solucionador más estable, en particular más estable para matrices singulares que cholesky a costa de ser más lento.
  • cholesky: usa la función estándar scipy.linalg.solve para obtener una solución de forma cerrada.
  • sparse_cg: usa el solucionador de gradiente conjugado que se encuentra en scipy.sparse.linalg.cg. Como algoritmo iterativo, este solucionador es más apropiado que cholesky para datos a gran escala (posibilidad de configurar toly max_iter).
  • lsqr: usa la rutina dedicada de mínimos cuadrados regularizados scipy.sparse.linalg.lsqr. Es el más rápido y utiliza un procedimiento iterativo.
  • sag: usa un descenso de gradiente promedio estocástico, y ‘saga’ usa su versión mejorada e imparcial llamada SAGA. Ambos métodos también utilizan un procedimiento iterativo y, a menudo, son más rápidos que otros solucionadores cuando tanto n_samples como n_features son grandes. Tenga en cuenta que la convergencia rápida ‘sag’ y ‘saga’ solo está garantizada en entidades con aproximadamente la misma escala. Puede preprocesar los datos con un escalador de sklearn.preprocessing.
  • lbfgs: usa el algoritmo L-BFGS-B implementado en scipy.optimize.minimize. Solo se puede usar cuando positive es Verdadero.

SVM¶

  • kernel: {'linear', 'poly', 'rbf', 'sigmoid', 'precomputed'} Especifica el nucleo que usa el algoritmo. Si no se especifica se usa 'rbf' por defecto.
  • degree: int, default=3 Grado de la función kernel polinomial ('poly'). Debe ser no negativo. Es ignorado por todos los demás núcleos.
  • tol: float, default=1e-3 Tolerancia para el criterio de parada.
  • C: float, default=1.0 Parámetro de regularización. La fuerza de la regularización es inversamente proporcional a C. Debe ser estrictamente positiva. La penalización es una penalización de l2 al cuadrado.
  • epsilon: float, default=0.1 Epsilon en el modelo epsilon-SVR. Especifica el tubo épsilon dentro del cual no se asocia ninguna penalización en la función de pérdida de entrenamiento con puntos predichos dentro de una distancia épsilon del valor real. Debe ser no negativo.

DecisionTreeRegressor¶

  • max_depth: profundidad máxima que puede alcanzar el árbol.

  • max_leaf_nodes: número máximo de nodos terminales.

RandomForestRegressor¶

  • n_estimators: int, default=100. Número de árboles en el bosque.
  • criterion: {'squared_error', 'absolute_error', 'friedman_mse', 'poisson'}, default='squared_error'. La función para medir la calidad de una división. Los criterios admitidos son:
    • 'squared_error' para el error cuadrático medio, que es igual a la reducción de la varianza como criterio de selección de características y minimiza la pérdida de L2 utilizando la media de cada nodo terminal,
    • 'friedman_mse', que utiliza el error cuadrático medio con la puntuación de mejora de Friedman para el potencial splits,
    • 'absolute_error' para el error absoluto medio, que minimiza la pérdida L1 usando la mediana de cada nodo terminal, y
    • 'poisson' que usa la reducción en la desviación de Poisson para encontrar divisiones. El entrenamiento con 'absolute_error' es significativamente más lento que cuando se usa 'squared_error'.
  • max_depth: int, default=None. La profundidad máxima del árbol. Si es None, los nodos se expanden hasta que todas las hojas sean puras o hasta que todas las hojas contengan menos de min_samples_split samples.
  • min_samples_split: int o float, default=2. El número mínimo de muestras requeridas para dividir un nodo interno.

GradientBoostingRegressor¶

  • n_estimators: int, default=100. El número de etapas de refuerzo a realizar. El descenso del gradiente es bastante resistente al sobreajuste, por lo que un gran número generalmente da como resultado un mejor rendimiento. Los valores deben estar en el rango [1, inf)
  • min_samples_leaf: int o float, Por defecto = 1. El número mínimo de muestras requeridas para estar en un nodo hoja. Un punto de división a cualquier profundidad solo se considerará si deja al menos min_sampleser el modelo.
  • max_depth: int o None, default = 3. Profundidad máxima de los estimadores de regresión individuales. La profundidad máxima limita el número de nodos en el árbol. Ajuste este parámetro para obtener el mejor rendimiento; el mejor valor depende de la interacción de las variables predictoras. Si es None, los nodos se expanden hasta que todas las hojas sean puras o hasta que todas las hojas contengan menos de min_samples_split samples. Si es int, los valores deben estar en el rango [1, inf).

# Ahora sí, predecimos
pred = model.predict(K_test)
In [59]:
def errores_svr_cumputed(y_test, pred):
    rmse = np.sqrt(np.mean((y_test - pred) ** 2))
    mae = np.mean(np.abs(y_test - pred))
    er = np.sum(np.abs(y_test - pred)) / np.sum(np.abs(y_test))
    
    errores = [{
        'RMSE': rmse,
        'MAE': mae,
        'ER': er
    }]
    
    # Retornamos el DataFrame (tendrá el índice 0 automático, pero no uno nombrado)
    return pd.DataFrame(errores)
